# 演示视频脚本（≤ 5 分钟）

> 录制工具：OBS Studio / Bandicam / ScreenToGif
> 导出格式：mp4，1080p，≤ 200MB
> 命名：`张三+李四+王五_演示.mp4`

---

## 时间线与台词

### 0:00-0:20 | 环境介绍

**画面**：VMware Workstation 窗口，OpenWrt 启动后的命令行界面

**台词**：
> 本项目基于 OpenWrt 24.10.0，部署在 VMware Workstation 17 虚拟机上，使用 NAT 网络模式。宿主机通过 SSH 远程管理。（画面切到 Windows 终端，展示 SSH 连接成功）

---

### 0:20-0:50 | Web 应用首页

**画面**：Windows Chrome 浏览器打开流量监控页面

**台词**：
> 打开浏览器，进入流量监控页面。上方是实时折线图，显示各活跃流的 avg2s 速率；下方是详细流量表格，展示每个五元组连接的累计字节和滑动窗口均值。

---

### 0:50-2:00 | 流量监控演示

**画面**：
1. 在终端执行 `ping 8.8.8.8` 持续发送 ICMP 包
2. 切回浏览器，折线图出现 ICMP 曲线实时跳动
3. 展示表格中对应的 ICMP 条目

**台词**：
> 现在在 VM 上执行持续 ping，模拟网络流量。可以看到折线图立即出现了一条新的 ICMP 流量曲线。停止 ping 后，曲线在约 5 秒内自动消失——这是活跃流过滤机制，只展示当前正在通信的连接。

---

### 2:00-3:30 | 防火墙配置演示

**画面**：
1. 点击导航切换到"防火墙配置"页
2. 表单填写：协议=TCP、源IP=0.0.0.0/0、目标IP=192.168.135.100、端口=80、动作=drop
3. 点击"添加规则"
4. 展示执行结果面板（stdout/stderr/code）
5. 在终端执行 `curl http://192.168.135.100` 验证被拒绝
6. 删除规则
7. 再次 `curl` 验证恢复

**台词**：
> 切换到防火墙配置页。添加一条规则：丢弃所有发往本机 80 端口的 TCP 流量。提交后可以看到执行结果——stdout 显示 fw4 命令的输出，code=0 表示成功。验证规则生效：在终端 curl 访问 OpenWrt，连接被拒绝。删除规则后再次访问，连接恢复。整个过程 stdout/stderr/exit code 完整展示，符合指导书 3.3 节要求。

---

### 3:30-4:30 | 后台架构展示

**画面**：
1. SSH 终端展示 `ps | grep traffic_monitor`——C 程序正在运行
2. `cat /tmp/traffic.json | head -20`——JSON 实时更新
3. 展示 Flask 后端进程
4. `nft list ruleset` 查看真实 nftables 规则

**台词**：
> 后台有三个关键组件：C 语言的 libpcap 流量监控程序，每秒将统计结果写入 /tmp/traffic.json；Python Flask 后端读取 JSON 并提供 REST API；防火墙操作直接调用 fw4/nftables shell 脚本。nft list ruleset 可以验证 Linux 内核中真实生效的防火墙规则。

---

### 4:30-5:00 | 总结

**画面**：回到 Web 首页

**台词**：
> 本项目实现了两个 OpenWrt 网络管理 Web 工具：基于 libpcap 的实时流量监控和基于 fw4 的防火墙配置管理。前后端分离架构，C 程序负责高性能数据采集，Python Flask 提供 REST API，Vue 3 前端实现可视化交互。谢谢观看。

1. **全程录屏**，不要有暂停或剪辑跳帧

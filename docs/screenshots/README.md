# 截图清单

> 截图存入对应子目录，用于实验报告。

---

## 需要截的图（按报告顺序）

### 1. 系统环境（1-2 张）

| # | 内容 | 怎么截 |
|---|---|---|
| 1.1 | OpenWrt 版本信息 | VM 控制台显示启动 banner（`OpenWrt 24.10.0`） |
| 1.2 | 网络配置 | `ip a` 输出，能看到 `br-lan` 的 IP `192.168.135.100` |

### 2. 流量监控（3-4 张）

| # | 内容 | 怎么截 |
|---|---|---|
| 2.1 | 流量监控页全貌 | 浏览器打开 `http://localhost:5175/`，切到流量监控页 |
| 2.2 | 折线图有数据 | 先在 VM 跑 `ping 8.8.8.8`，然后截折线图显示 ICMP 曲线 |
| 2.3 | 详细表格 | 展示活跃流排在最前面的表格 |
| 2.4 | C 程序运行 | 终端 `ps \| grep traffic_monitor` + `cat /tmp/traffic.json \| head -20` |

### 3. 防火墙配置（4-5 张）

| # | 内容 | 怎么截 |
|---|---|---|
| 3.1 | 防火墙配置页 | 浏览器切到防火墙页，表单初始状态 |
| 3.2 | 添加规则 | 表单填完（tcp, drop port 80），点击提交后的结果面板 |
| 3.3 | **规则生效前**（指导书 3.3 硬要求） | Windows 终端执行 `curl http://192.168.135.100`，返回 LuCI 页面 HTML |
| 3.4 | **规则生效后**（指导书 3.3 硬要求） | 同一终端 `curl --max-time 3 http://192.168.135.100`，超时或被拒 |
| 3.5 | **规则删除后恢复**（指导书 3.3 硬要求） | 删除规则后再 `curl`，恢复访问 |
| 3.6 | nftables 真实规则 | VM 终端 `nft list ruleset` 输出 |

### 4. 后端（1-2 张）

| # | 内容 | 怎么截 |
|---|---|---|
| 4.1 | Flask 后端运行 | 终端 `ps \| grep python3` + `curl http://localhost:5000/api/health` 返回 `mockMode:false` |
| 4.2 | 防火墙测试通过 | 终端 `pytest test/ -v` 输出（全部 pass） |

### 5. 防火墙验证截图（存 `firewall-verify/`）

| # | 内容 |
|---|---|
| 5.1 | 添加规则前 curl 可访问 |
| 5.2 | 添加规则后 curl 不可访问 |
| 5.3 | 删除规则后 curl 恢复访问 |
| 5.4 | `fw4 print` 或 `nft list ruleset` 显示规则已生效 |

---

## 截图要求

- **分辨率**：1920x1080 或更高
- **格式**：PNG（无损）或 JPG（质量 90%+）
- **命名**：`01-openwrt-version.png`、`02-traffic-page.png`... 编号对应报告插入顺序
- **不要有个人信息**（书签、桌面图标、聊天弹窗）
